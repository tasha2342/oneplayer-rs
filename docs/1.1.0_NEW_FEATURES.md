# OnePlayer 1.1.0

## RTB 스케줄

- 기존 `play_data`의 `slots`를 유지하면서 별도 `rtb_slots` 구간을 지원한다.
- RTB 구간에서는 MP4 영상과 JPEG/PNG 이미지를 전체 화면으로 표출한다.
- RTB 에셋 다운로드 또는 prepare 실패 시 같은 절대 시각의 일반 편성으로
  자동 복귀한다.
- 백엔드 계약과 완성 예제는
  [`api/play_data_v1.1.0.md`](api/play_data_v1.1.0.md)를 참고한다.
- 플레이어 내부 병합·캐시·fallback 규칙은 [`RTB.md`](RTB.md)를 참고한다.

## Tracking URL

- `impression`, `start`, `firstquartile`, `midpoint`, `thirdquartile`,
  `complete` 이벤트를 지원한다.
- URL 호출은 렌더링과 분리된 bounded 비동기 큐에서 수행한다.
- HTTP GET, 5초 타임아웃, 최대 3회 총 시도와 지수 백오프를 적용한다.
- 앱 정상 종료 시 큐를 drain하며 동일 장면의 같은 이벤트 URL은 중복 호출하지
  않는다.
- 이벤트별 정확한 발생 시점은 [`TRACKING_URL.md`](TRACKING_URL.md)를
  참고한다.
